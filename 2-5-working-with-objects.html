What You'll Learn

Everything is an Object

Unlike traditional shells that pass text between commands, PowerShell passes objects. This is PowerShell's most important concept. Objects have properties (data) and methods (actions), making them far more powerful than simple text strings.

Memory Aid: Key Object Commands
  • Get-Member (gm) ‚Üí Discover properties and methods
  • Select-Object ‚Üí Choose which properties to display
  • Where-Object ‚Üí Filter objects by criteria
  • Sort-Object ‚Üí Order objects by property
  • ForEach-Object ‚Üí Process each object
  • Measure-Object ‚Üí Calculate statistics

Exploring Objects with Get-Member

üî∑ PowerShell - Get-Member 
# What type of object does Get-Process return?
Get-Process | Get-Member
# TypeName: System.Diagnostics.Process

# Show only properties
Get-Process | Get-Member -MemberType Property

# Show only methods
Get-Process | Get-Member -MemberType Method

# Common alias: gm
Get-Service | gm

# See all properties of a specific object
$Process = Get-Process -Name "explorer" | Select-Object -First 1
$Process | Format-List *

# Get object type
$Process.GetType()
$Process.GetType().FullName  # System.Diagnostics.Process

# Check what's available on services
Get-Service "WinRM" | Get-Member

# See methods you can call
Get-Service "Spooler" | Get-Member -MemberType Method
# Start(), Stop(), Pause(), Refresh()...

Accessing Properties

üî∑ PowerShell - Property Access 
# Dot notation for properties
$Service = Get-Service "WinRM"
$Service.Name          # WinRM
$Service.Status        # Running
$Service.StartType     # Automatic
$Service.DisplayName   # Windows Remote Management (WS-Management)

# Access property on collection (PS 3+)
(Get-Process).Name           # All process names
(Get-Service).Status         # All service statuses

# Dynamic property access with variable
$PropertyName = "Status"
$Service.$PropertyName       # Running

# Nested properties
$Process = Get-Process "explorer" | Select-Object -First 1
$Process.StartTime.DayOfWeek
$Process.Modules.Count

# Check if property exists
$Process.PSObject.Properties.Name -contains 'CPU'

Select-Object - Choosing Properties

üî∑ PowerShell - Select-Object 
# Select specific properties
Get-Process | Select-Object Name, Id, CPU

# First/Last N objects
Get-Process | Select-Object -First 5
Get-Process | Select-Object -Last 3
Get-Process | Select-Object -Skip 10 -First 5

# Unique values
Get-Process | Select-Object Name -Unique

# Exclude properties
Get-Process | Select-Object * -ExcludeProperty Modules, Threads

# Calculated properties (custom columns)
Get-Process | Select-Object Name, 
    @{Name='Memory(MB)'; Expression={[math]::Round($_.WorkingSet64 / 1MB, 2)}},
    @{N='CPU(s)'; E={[math]::Round($_.CPU, 2)}}

# Rename property
Get-Service | Select-Object @{N='ServiceName';E={$_.Name}}, Status

# ExpandProperty - get raw value instead of object
Get-Process | Select-Object -ExpandProperty Name
# vs
Get-Process | Select-Object Name  # Returns objects with Name property

# Common alias: select
Get-Process | select Name, CPU -First 5

Where-Object - Filtering

üî∑ PowerShell - Where-Object 
# Filter by property value
Get-Service | Where-Object Status -eq 'Running'

# Script block syntax (more flexible)
Get-Service | Where-Object { $_.Status -eq 'Running' }

# Multiple conditions
Get-Process | Where-Object { $_.CPU -gt 10 -and $_.WorkingSet64 -gt 100MB }

# Or conditions
Get-Service | Where-Object { $_.Name -eq 'WinRM' -or $_.Name -eq 'Spooler' }

# Pattern matching
Get-Service | Where-Object Name -like 'Win*'
Get-Service | Where-Object Name -match '^Win'

# Not equal / Not like
Get-Process | Where-Object Name -ne 'svchost'
Get-Service | Where-Object Name -notlike '*Update*'

# In / NotIn array
$CriticalServices = @('WinRM', 'W32Time', 'Netlogon')
Get-Service | Where-Object Name -in $CriticalServices

# Comparison operators: -gt -lt -ge -le -eq -ne
Get-Process | Where-Object { $_.Handles -ge 1000 }

# Common alias: where, ?
Get-Process | ? CPU -gt 5

Sort-Object - Ordering

üî∑ PowerShell - Sort-Object 
# Sort ascending (default)
Get-Process | Sort-Object CPU

# Sort descending
Get-Process | Sort-Object CPU -Descending

# Sort by multiple properties
Get-Process | Sort-Object Name, CPU -Descending

# Sort with different directions per property
Get-Process | Sort-Object @{Expression='Name';Ascending=$true}, 
                           @{Expression='CPU';Descending=$true}

# Top 10 by CPU
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10

# Unique values only
Get-Process | Sort-Object Name -Unique

# Case-sensitive sort
'Apple', 'apple', 'Banana' | Sort-Object -CaseSensitive

# Sort by calculated property
Get-ChildItem | Sort-Object { $_.Length / 1KB } -Descending

# Common alias: sort
Get-Process | sort CPU -desc

Creating Custom Objects

üî∑ PowerShell - Custom Objects 
# Method 1: [PSCustomObject] (preferred, PS 3+)
$Server = [PSCustomObject]@{
    Name = 'Server01'
    IP = '192.168.1.10'
    Status = 'Online'
    LastCheck = Get-Date
}

# Method 2: New-Object with Add-Member
$Server = New-Object PSObject
$Server | Add-Member -NotePropertyName 'Name' -NotePropertyValue 'Server01'
$Server | Add-Member -NotePropertyName 'IP' -NotePropertyValue '192.168.1.10'

# Creating collection of objects
$Servers = @(
    [PSCustomObject]@{ Name = 'Server01'; Role = 'Web'; Status = 'Online' }
    [PSCustomObject]@{ Name = 'Server02'; Role = 'DB'; Status = 'Online' }
    [PSCustomObject]@{ Name = 'Server03'; Role = 'App'; Status = 'Offline' }
)

# Build objects in loop
$Results = Get-Process | ForEach-Object {
    [PSCustomObject]@{
        ProcessName = $_.Name
        PID = $_.Id
        'Memory(MB)' = [math]::Round($_.WorkingSet64 / 1MB, 2)
        'CPU(s)' = [math]::Round($_.CPU, 2)
        Running = $_.Responding
    }
}

# Security report example
$SecurityReport = Get-EventLog -LogName Security -Newest 100 |
    Group-Object EventID |
    ForEach-Object {
        [PSCustomObject]@{
            EventID = $_.Name
            Count = $_.Count
            Description = ($_.Group | Select-Object -First 1).Message.Split('.')[0]
        }
    } | Sort-Object Count -Descending

Measure-Object and Group-Object

üî∑ PowerShell - Aggregation 
# Measure-Object - statistics
Get-Process | Measure-Object                          # Count
Get-Process | Measure-Object -Property CPU -Sum       # Sum
Get-Process | Measure-Object -Property CPU -Average   # Average
Get-Process | Measure-Object CPU -Sum -Average -Maximum -Minimum

# File sizes
Get-ChildItem -Recurse | Measure-Object -Property Length -Sum

# Group-Object - categorize
Get-Service | Group-Object Status

# Group and count
Get-Process | Group-Object Name | 
    Sort-Object Count -Descending | 
    Select-Object Name, Count -First 10

# Group events by source
Get-EventLog System -Newest 500 | 
    Group-Object Source | 
    Sort-Object Count -Descending

# Group as hashtable for quick lookup
$ServicesByStatus = Get-Service | Group-Object Status -AsHashTable
$ServicesByStatus['Running'].Count

# No element (just count)
Get-Process | Group-Object Name -NoElement

Best Practices: Working with Objects
  • Always explore with Get-Member: Discover available properties/methods
  • Filter early: Use Where-Object before expensive operations
  • Select only needed properties: Reduces memory and improves speed
  • Use [PSCustomObject] for reports: Clean, exportable output
  • Avoid text parsing: Access properties directly instead
  • Use -ExpandProperty for single values: Get raw value, not object
üíº
Interview Tips
  • Common Question: "How is PowerShell different from Bash?"
  • Strong Answer: PowerShell passes objects between commands, not text. This means you can access properties directly ($process.Name) instead of parsing text with grep/awk. Use Get-Member to discover what properties and methods are available on any object.
  • Follow-up: Mention that calculated properties in Select-Object let you transform data on the fly, and PSCustomObject creates clean reports.

Key Takeaways

  • PowerShell passes objects, not text‚Äîaccess properties directly
  • Use Get-Member to explore any object's properties and methods
  • Select-Object chooses properties, Where-Object filters
  • [PSCustomObject] creates clean, exportable reports
  • Calculated properties transform data inline: @{N='Name';E={expression}}
  • Measure-Object and Group-Object aggregate data
