Variable Basics

üî∑ PowerShell - Creating Variables 
# Simple assignment
$Name = "Server01"
$Count = 42
$IsEnabled = $true

# Check variable type
$Name.GetType().Name     # String
$Count.GetType().Name    # Int32

# Type-constrained variable
[string]$Server = "DC01"
[int]$Port = 443
[datetime]$StartDate = "2024-01-15"

# Variable in strings (expansion)
$Message = "Connecting to $Server on port $Port"

# No expansion with single quotes
$Literal = 'Variable is $Server'  # Outputs: Variable is $Server

# Complex expressions in strings
$Info = "Total processes: $((Get-Process).Count)"

# Clear/remove variable
Remove-Variable -Name Count
$Server = $null

Common Data Types

üî∑ PowerShell - Data Types 
# String
[string]$Text = "Hello World"
$Text.Length                        # 11
$Text.ToUpper()                     # HELLO WORLD
$Text.Replace("World", "PowerShell") # Hello PowerShell

# Integer
[int]$Number = 100
[int64]$BigNumber = 9999999999

# Boolean
[bool]$Flag = $true
$Flag = $false

# DateTime
[datetime]$Now = Get-Date
[datetime]$Custom = "2024-06-15 14:30:00"
$Now.AddDays(7)                      # Add 7 days
$Now.ToString("yyyy-MM-dd")          # Format output

# Double (decimal)
[double]$Price = 19.99

# Type casting
[int]"42"                            # String to int
[string]100                          # Int to string
[datetime]"2024-01-01"              # String to datetime

Arrays

üî∑ PowerShell - Arrays 
# Create array
$Servers = @("DC01", "DC02", "WEB01", "SQL01")

# Alternative syntax
$Numbers = 1, 2, 3, 4, 5

# Range operator
$Range = 1..10

# Access elements (0-indexed)
$Servers[0]                        # DC01 (first)
$Servers[-1]                       # SQL01 (last)
$Servers[0..1]                     # DC01, DC02 (slice)

# Array properties
$Servers.Count                     # 4
$Servers.Length                    # 4

# Add to array (creates new array!)
$Servers += "APP01"

# Better: Use ArrayList for frequent additions
$List = [System.Collections.ArrayList]@()
[void]$List.Add("Server1")
[void]$List.Add("Server2")

# Check if contains
$Servers -contains "DC01"          # True
"DC01" -in $Servers                # True

# Loop through array
foreach ($Server in $Servers) {
    Write-Host "Processing: $Server"
}

# Filter array
$DCServers = $Servers | Where-Object { $_ -like "DC*" }

Hashtables

üî∑ PowerShell - Hashtables 
# Create hashtable
$ServerInfo = @{
    Name = "DC01"
    IP = "192.168.1.10"
    Role = "Domain Controller"
    Port = 389
}

# Access values
$ServerInfo["Name"]               # DC01
$ServerInfo.IP                    # 192.168.1.10

# Add/modify keys
$ServerInfo["OS"] = "Windows Server 2022"
$ServerInfo.Location = "DataCenter1"

# Remove key
$ServerInfo.Remove("Port")

# Check for key
$ServerInfo.ContainsKey("Name")    # True

# Get all keys/values
$ServerInfo.Keys
$ServerInfo.Values

# Loop through hashtable
foreach ($Key in $ServerInfo.Keys) {
    Write-Host "$Key : $($ServerInfo[$Key])"
}

# Ordered hashtable (maintains insertion order)
$OrderedHash = [ordered]@{
    First = 1
    Second = 2
    Third = 3
}

# Splatting with hashtables
$Params = @{
    ComputerName = "DC01"
    Credential = $Cred
    ScriptBlock = { Get-Service }
}
Invoke-Command @Params   # Splat with @

Automatic & Special Variables

üî∑ PowerShell - Special Variables 
# Automatic variables
$_              # Current pipeline object
$PSItem         # Same as $_
$null           # Null/empty value
$true           # Boolean true
$false          # Boolean false
$Error          # Array of recent errors
$?              # Success of last command
$LASTEXITCODE   # Exit code of last native command
$PWD            # Current directory
$HOME           # User's home directory
$Host           # Host application info
$PID            # Process ID of PowerShell
$PSVersionTable # PowerShell version info

# Environment variables
$env:USERNAME          # Current user
$env:COMPUTERNAME      # Computer name
$env:PATH              # System PATH
$env:TEMP              # Temp directory

# Set environment variable
$env:MY_VAR = "MyValue"

# Preference variables
$ErrorActionPreference = 'Stop'      # Stop on errors
$VerbosePreference = 'Continue'      # Show verbose output
$DebugPreference = 'Continue'        # Show debug output
$ConfirmPreference = 'None'          # Suppress confirmations

Best Practices: Variables
  • Use meaningful names: $ServerList not $s
  • PascalCase convention: $ComputerName
  • Type-constrain when needed: [int]$Port
  • Use ArrayList for growth: Avoid += on arrays
  • Prefer ordered hashtables: When order matters
  • Use splatting: For cleaner cmdlet calls
üíº
Interview Tips
  • Common Question: "What's the difference between @() and @{}?"
  • Strong Answer: @() creates an array‚Äîan ordered collection accessed by index (0, 1, 2). @{} creates a hashtable‚Äîa key-value collection accessed by key name. Arrays are best for lists of similar items; hashtables are best for configuration or properties with named values.

Key Takeaways

  • Variables start with $ and are dynamically typed
  • Use [type] to constrain variable types
  • Arrays (@()) are ordered, accessed by index
  • Hashtables (@{}) are key-value pairs
  • Use ArrayList instead of += for performance
  • Splatting (@Params) makes cmdlet calls cleaner
