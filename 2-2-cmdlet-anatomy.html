2.2 Cmdlet Anatomy

The Verb-Noun Convention

Every PowerShell cmdlet follows a consistent Verb-Noun naming pattern. This makes cmdlets discoverable and predictable.

PowerShell - Verb-Noun Structure 
# Structure: Verb-Noun
Get-Process       # Gets process information
Stop-Process      # Stops a process
Get-Service       # Gets service information
Start-Service     # Starts a service
Get-EventLog      # Gets event log entries
New-Item          # Creates a new item
Remove-Item       # Removes an item
Set-Location      # Changes current location
üß†
Memory Aid: Common Verbs

G-S-N-R-I: Get (retrieve), Set (modify), New (create), Remove (delete), Invoke (execute)

Approved Verbs

PowerShell has a set of approved verbs to ensure consistency. View them with:

PowerShell - View Approved Verbs 
# List all approved verbs
Get-Verb

# Group by category
Get-Verb | Group-Object Group | Select-Object Name, Count

# Common verb categories:
# - Common: Get, Set, New, Remove, Add, Clear, Close, Copy, Move
# - Data: Compare, Convert, Export, Import, Merge
# - Lifecycle: Start, Stop, Restart, Enable, Disable
# - Security: Block, Grant, Revoke, Protect, Unprotect
Verb Purpose Example
Get Retrieve data Get-Process, Get-ADUser
Set Modify existing data Set-ADUser, Set-Service
New Create new resource New-Item, New-ADUser
Remove Delete resource Remove-Item, Remove-ADUser
Start Begin operation Start-Process, Start-Service
Stop End operation Stop-Process, Stop-Service
Invoke Execute action Invoke-Command, Invoke-WebRequest
Export Output to file Export-Csv, Export-Clixml
Import Read from file Import-Csv, Import-Module

Understanding Parameters

Parameters allow you to customize cmdlet behavior. There are several types:

Named Parameters

PowerShell - Named Parameters 
# Named parameters use -ParameterName Value format
Get-Process -Name "notepad"
Get-Service -Name "WinRM" -ComputerName "Server01"
Get-EventLog -LogName "Security" -Newest 10

# Parameter names can be shortened (if unambiguous)
Get-Process -N "notepad"    # -N is short for -Name
Get-EventLog -Log Security  # -Log is short for -LogName

Positional Parameters

PowerShell - Positional Parameters 
# Some parameters accept values by position (without -Name)
Get-Process notepad              # Same as Get-Process -Name notepad
Get-ChildItem C:\Windows         # Same as Get-ChildItem -Path C:\Windows
Get-Content C:\logs\app.log      # Same as Get-Content -Path C:\logs\app.log

# Check which parameters are positional with Get-Help
Get-Help Get-Process -Parameter Name
# Shows: Position? 0 (first positional parameter)

Switch Parameters

PowerShell - Switch Parameters 
# Switch parameters are boolean flags - just include them to enable
Get-ChildItem -Recurse              # Enable recursive listing
Get-Process -IncludeUserName        # Include username in output
Remove-Item -Force                  # Force deletion
Get-ChildItem -Hidden               # Show hidden files

# Explicitly set switch value (rarely needed)
Get-ChildItem -Recurse:$true
Get-ChildItem -Recurse:$false

Common Parameters

All cmdlets support a set of common parameters:

PowerShell - Common Parameters 
# -Verbose: Show detailed operation info
Get-Process -Verbose

# -WhatIf: Preview what would happen (no actual changes)
Stop-Process -Name notepad -WhatIf

# -Confirm: Ask for confirmation before action
Remove-Item *.log -Confirm

# -ErrorAction: Control error behavior
Get-Process -Name "nonexistent" -ErrorAction SilentlyContinue
Get-Process -Name "nonexistent" -ErrorAction Stop

# -OutVariable: Store output in variable while displaying
Get-Process | Where-Object CPU -gt 100 -OutVariable highCPU

# Common ErrorAction values:
# Continue (default), SilentlyContinue, Stop, Ignore, Inquire
‚úÖ
Best Practice

Always use -WhatIf when running destructive commands like Remove-Item, Stop-Process, or Set-* cmdlets for the first time. This shows what would happen without making changes.

Getting Help

Get-Help is your most important tool for learning cmdlets.

PowerShell - Using Get-Help 
# Update help files (run as admin, do this first!)
Update-Help -Force -ErrorAction SilentlyContinue

# Basic help
Get-Help Get-Process

# Detailed help with examples
Get-Help Get-Process -Detailed

# Full help (everything)
Get-Help Get-Process -Full

# Just the examples
Get-Help Get-Process -Examples

# Help for a specific parameter
Get-Help Get-Process -Parameter Name

# Online help (opens in browser)
Get-Help Get-Process -Online

# Wildcard search for cmdlets
Get-Help *process*
Get-Help *event*log*

Discovering Cmdlets with Get-Command

PowerShell - Using Get-Command 
# Find all cmdlets with a specific verb
Get-Command -Verb Get
Get-Command -Verb Remove
Get-Command -Verb Export

# Find all cmdlets for a noun
Get-Command -Noun Process
Get-Command -Noun Service
Get-Command -Noun *User*

# Combine verb and noun
Get-Command -Verb Get -Noun *Event*

# Find cmdlets in a module
Get-Command -Module ActiveDirectory
Get-Command -Module Microsoft.PowerShell.Security

# Find cmdlets that work with a parameter
Get-Command -ParameterName ComputerName

# Show cmdlet source module
Get-Command Get-Process | Select-Object Name, Source, Version

Aliases

Aliases are shortcuts for cmdlets. Know them but prefer full names in scripts.

PowerShell - Common Aliases 
# View all aliases
Get-Alias

# Find alias for a cmdlet
Get-Alias -Definition Get-ChildItem    # Returns: dir, gci, ls

# Find what an alias refers to
Get-Alias ls                           # Returns: Get-ChildItem
Get-Alias gc                           # Returns: Get-Content

# Common aliases (Unix-like)
ls      # Get-ChildItem (list directory)
cd      # Set-Location (change directory)
pwd     # Get-Location (print working directory)
cat     # Get-Content (display file contents)
rm      # Remove-Item (delete)
cp      # Copy-Item (copy)
mv      # Move-Item (move)
ps      # Get-Process (process list)

# Common aliases (CMD-like)
dir     # Get-ChildItem
cls     # Clear-Host
copy    # Copy-Item
del     # Remove-Item
type    # Get-Content

# PowerShell-specific short forms
gci     # Get-ChildItem
gcm     # Get-Command
gps     # Get-Process
gsv     # Get-Service
iwr     # Invoke-WebRequest
icm     # Invoke-Command
‚ö†Ô∏è
Warning

Never use aliases in scripts - always use full cmdlet names. Aliases are for interactive use only. They may not exist on all systems and make scripts harder to read.

Tab Completion

PowerShell - Tab Completion Tips 
# Tab completion for cmdlets
Get-Pro[TAB]                    # Completes to Get-Process
Get-Event[TAB][TAB]             # Cycles through Get-Event*

# Tab completion for parameters
Get-Process -[TAB]              # Shows available parameters
Get-Process -N[TAB]             # Completes to -Name

# Tab completion for parameter values
Get-Service -Name wi[TAB]       # Completes service names starting with "wi"
Get-Process -Name note[TAB]     # Completes to notepad if running

# Tab completion for paths
Get-ChildItem C:\Win[TAB]       # Completes to C:\Windows\
Get-Content .\log[TAB]          # Completes local file names

# Ctrl+Space shows all options (in ISE/VSCode)
Get-Process -Name [Ctrl+Space]  # Shows all running process names

Exploring Objects with Get-Member

PowerShell - Using Get-Member 
# See what properties and methods an object has
Get-Process | Get-Member

# Just properties
Get-Process | Get-Member -MemberType Property

# Just methods
Get-Process | Get-Member -MemberType Method

# See the type name
Get-Process | Get-Member | Select-Object -First 1 TypeName

# Shorthand
Get-Process | gm

Interview Tip

When asked "How do you learn a new cmdlet?", mention the three discovery cmdlets: Get-Command (find cmdlets), Get-Help (learn syntax), and Get-Member (explore output objects). This shows methodical PowerShell proficiency.

Key Takeaways

  • Cmdlets use Verb-Noun naming: Get-Process, Set-Service
  • Parameters can be named (-Name), positional, or switch (-Recurse)
  • Use Get-Help with -Examples to learn any cmdlet
  • Use Get-Command to discover available cmdlets
  • Use Get-Member to explore object properties
  • Always use -WhatIf before destructive operations
  • Use full cmdlet names in scripts, aliases only interactively
