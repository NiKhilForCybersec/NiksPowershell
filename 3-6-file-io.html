3.6 File I/O

üìñ
What You'll Learn
  • Read files line by line and in bulk
  • Write and append to files
  • Use redirection operators effectively
  • Work with temporary files securely
üß†
Memory Aid: Redirection Operators
  • > ‚Üí Write to file (overwrite)
  • >> ‚Üí Append to file
  • < ‚Üí Read from file (input)
  • 2> ‚Üí Redirect stderr
  • &> ‚Üí Redirect both stdout and stderr
  • 2>&1 ‚Üí Redirect stderr to stdout
  • | ‚Üí Pipe stdout to next command

Reading Files

üêß Bash - Reading Files 
#!/bin/bash

# Display entire file
cat /etc/passwd

# Display with line numbers
cat -n /etc/passwd
nl /etc/passwd

# First/last lines
head -n 10 /var/log/syslog     # First 10 lines
tail -n 20 /var/log/syslog     # Last 20 lines
tail -f /var/log/syslog        # Follow (live updates)

# Read file into variable
content=$(cat config.txt)
echo "$content"

# Read file line by line (recommended method)
while IFS= read -r line; do
    echo "Line: $line"
done < /etc/hosts

# Read with line number
line_num=0
while IFS= read -r line; do
    ((line_num++))
    echo "$line_num: $line"
done < file.txt

# Read specific fields (CSV-like)
while IFS=':' read -r user pass uid gid gecos home shell; do
    echo "User: $user, Shell: $shell"
done < /etc/passwd

# Read into array
mapfile -t lines < file.txt
echo "Total lines: ${#lines[@]}"
echo "First line: ${lines[0]}"

# Alternative: readarray
readarray -t data < config.txt

# Read specific line
sed -n '5p' file.txt           # Line 5
sed -n '5,10p' file.txt        # Lines 5-10
awk 'NR==5' file.txt           # Line 5 with awk

Writing Files

üêß Bash - Writing Files 
#!/bin/bash

# Overwrite file
echo "Hello World" > output.txt

# Append to file
echo "Another line" >> output.txt

# Multi-line with heredoc
cat > config.yml <

# Heredoc with variable expansion
name="MyApp"
cat > settings.conf <

# Heredoc without variable expansion (quote delimiter)
cat > script.sh <<'EOF'
#!/bin/bash
echo "Current user: $USER"
echo "Home: $HOME"
EOF

# Write command output to file
date > timestamp.txt
ls -la > directory_listing.txt
ps aux > processes.txt

# Redirect both stdout and stderr
command &> all_output.txt
command > stdout.txt 2> stderr.txt

# Append stderr to same file as stdout
command >> output.log 2>&1

# Write from loop
for i in {1..10}; do
    echo "Line $i"
done > numbers.txt

# Use tee to write and display
echo "Important message" | tee output.txt
ls -la | tee listing.txt           # Write and show
ls -la | tee -a listing.txt        # Append and show

# Write to multiple files
echo "Log entry" | tee file1.txt file2.txt file3.txt

# printf for formatted output
printf "%-20s %10s\n" "Name" "Value" > report.txt
printf "%-20s %10d\n" "Count" 42 >> report.txt

File Descriptors

üêß Bash - File Descriptors 
#!/bin/bash

# Standard file descriptors:
# 0 = stdin (input)
# 1 = stdout (output)
# 2 = stderr (errors)

# Redirect stderr only
ls /nonexistent 2> errors.txt

# Discard stderr
ls /nonexistent 2>/dev/null

# Redirect stderr to stdout
ls /nonexistent 2>&1

# Discard all output
command &>/dev/null

# Open custom file descriptor for reading
exec 3< input.txt
while read -r line <&3; do
    echo "$line"
done
exec 3<&-  # Close descriptor

# Open custom file descriptor for writing
exec 4> output.txt
echo "Writing via FD 4" >&4
echo "More content" >&4
exec 4>&-  # Close descriptor

# Read and write simultaneously
exec 3< input.txt
exec 4> output.txt
while read -r line <&3; do
    echo "Processed: $line" >&4
done
exec 3<&- 4>&-

# Swap stdout and stderr
command 3>&1 1>&2 2>&3

Temporary Files

üêß Bash - Temporary Files 
#!/bin/bash

# Create secure temp file
tmpfile=$(mktemp)
echo "Temp file: $tmpfile"

# Create temp file with custom prefix
tmpfile=$(mktemp /tmp/myapp.XXXXXX)

# Create temp directory
tmpdir=$(mktemp -d)
echo "Temp dir: $tmpdir"

# Cleanup on exit (CRITICAL for security)
cleanup() {
    rm -f "$tmpfile"
    rm -rf "$tmpdir"
}
trap cleanup EXIT

# Use temp file
echo "Processing data..." > "$tmpfile"
cat "$tmpfile"

# Secure temp file pattern
set -e  # Exit on error

tmpfile=$(mktemp) || exit 1
trap 'rm -f "$tmpfile"' EXIT

# Process data securely
chmod 600 "$tmpfile"  # Restrict permissions
sensitive_data > "$tmpfile"

# Named pipe (FIFO) for inter-process communication
pipe=$(mktemp -u)
mkfifo "$pipe"
trap 'rm -f "$pipe"' EXIT

# Process substitution (no temp file needed)
diff <(ls dir1) <(ls dir2)
comm <(sort file1) <(sort file2)

Common File Operations

üêß Bash - File Operations 
#!/bin/bash

# Check if file exists before reading
if [[ -f "config.txt" ]]; then
    cat config.txt
else
    echo "Config file not found"
    exit 1
fi

# Check if file is readable
if [[ -r "$file" ]]; then
    cat "$file"
fi

# Check if file is writable
if [[ -w "$file" ]]; then
    echo "data" >> "$file"
fi

# Check if file is empty
if [[ ! -s "$file" ]]; then
    echo "File is empty"
fi

# Get file size
size=$(stat -c%s "$file")            # Linux
size=$(stat -f%z "$file")            # macOS
size=$(wc -c < "$file")              # Portable

# Count lines
lines=$(wc -l < "$file")

# Get file modification time
mod_time=$(stat -c%Y "$file")        # Epoch seconds

# Touch file (create or update timestamp)
touch newfile.txt

# Copy file
cp source.txt dest.txt
cp -r source_dir/ dest_dir/         # Recursive

# Move/rename
mv old.txt new.txt

# Delete
rm file.txt
rm -rf directory/                    # Recursive, force

# Create directory
mkdir -p /path/to/nested/dir        # Create parents

# Find files
find /var/log -name "*.log" -mtime -1  # Modified in last day
find . -type f -size +100M           # Files > 100MB

Security Log Processing Example

üêß Bash - Log Processing Script 
#!/bin/bash
# Process auth log and report failed logins

set -euo pipefail

LOG_FILE="/var/log/auth.log"
OUTPUT_FILE="failed_logins_$(date +%Y%m%d).csv"

# Check log exists
if [[ ! -r "$LOG_FILE" ]]; then
    echo "Error: Cannot read $LOG_FILE" >&2
    exit 1
fi

# Create CSV header
echo "Timestamp,User,Source_IP,Count" > "$OUTPUT_FILE"

# Extract failed login attempts
grep "Failed password" "$LOG_FILE" | \
    awk '{
        for(i=1; i<=NF; i++) {
            if($i == "for") user=$(i+1)
            if($i == "from") ip=$(i+1)
        }
        print user "," ip
    }' | \
    sort | uniq -c | \
    awk '{print strftime("%Y-%m-%d %H:%M:%S") "," $2 "," $1}' >> "$OUTPUT_FILE"

# Summary
total=$(tail -n +2 "$OUTPUT_FILE" | wc -l)
echo "Processed $total unique failed login combinations"
echo "Report saved to: $OUTPUT_FILE"

# Show top offenders
echo ""
echo "Top 10 source IPs:"
grep "Failed password" "$LOG_FILE" | \
    grep -oE 'from [0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' | \
    cut -d' ' -f2 | \
    sort | uniq -c | \
    sort -rn | \
    head -10

Best Practices: File I/O
  • Use read -r: Prevents backslash interpretation
  • Set IFS= for read: Preserves leading/trailing whitespace
  • Always quote file paths: Handles spaces in filenames
  • Use mktemp for temp files: Secure, unique names
  • Trap EXIT for cleanup: Remove temp files on exit
  • Check file existence: Before reading with [[ -f ]]
üíº
Interview Tips
  • Common Question: "How do you read a file line by line in Bash?"
  • Strong Answer: Use while IFS= read -r line; do ... done < file. The IFS= preserves whitespace, -r prevents backslash interpretation. Avoid for line in $(cat file) as it breaks on whitespace and is slower.

Key Takeaways

  • > overwrites, >> appends, < reads input
  • Use while IFS= read -r for proper line reading
  • 2> redirects stderr, &> redirects both streams
  • Create temp files with mktemp, clean up with trap EXIT
  • Use heredocs (<) for multi-line content
  • tee writes to file and displays output simultaneously
