What is Bash?

Bash (Bourne Again Shell) is the default command-line interpreter for most Linux distributions and macOS. Created in 1989 as a free replacement for the Bourne shell (sh), Bash has become the de facto standard for Unix shell scripting.

Unlike PowerShell which passes objects through pipelines, Bash follows the traditional Unix philosophy of treating everything as text streams. This makes it incredibly powerful for text processing, log analysis, and working with command-line tools.

Best Practice: Script Portability
  • Use #!/bin/bash for scripts requiring Bash features
  • Use #!/bin/sh for maximum portability (POSIX only)
  • Avoid bashisms if your script must run on minimal systems (Docker, busybox)
  • Test on target systems before deploymentâ€”shells differ subtly

# Print output

# Exit with success

# Count login attempts by IP
grep "Failed password" /var/log/auth.log | \
    grep -oE '[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' | \

Interview Tips
  • Common Question: "What's the difference between Bash and sh?"
  • Strong Answer: Bash is a superset of sh (POSIX shell). Bash adds features like arrays, [[ ]] tests, and brace expansion. Scripts using #!/bin/sh should be POSIX-compatible, while #!/bin/bash can use Bash-specific features.
  • Follow-up: "When would you use sh over Bash?" â†’ For maximum portability, minimal Docker images, or systems where Bash isn't installed.
  • Bonus: Mention that you test scripts on both for compatibility

  • Bash is the default shell on most Linux distributionsâ€”essential for security work
  • Unix philosophy: small tools that do one thing well, chained via pipes
  • Bash pipelines pass text streams, unlike PowerShell's object pipeline
  • Use #!/bin/bash for Bash features, #!/bin/sh for maximum portability
