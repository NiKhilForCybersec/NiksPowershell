2.8 Working with Files

What You'll Learn

Memory Aid: File Cmdlets
  • Get-Content ‚Üí Read file (alias: gc, cat, type)
  • Set-Content ‚Üí Write/overwrite file
  • Add-Content ‚Üí Append to file
  • Out-File ‚Üí Redirect output to file
  • Get-ChildItem ‚Üí List files (alias: dir, ls)
  • Test-Path ‚Üí Check if path exists

Reading Files

üî∑ PowerShell - Reading Files 
# Read entire file as array of lines
$Lines = Get-Content "C:\Logs\app.log"
$Lines.Count                              # Number of lines
$Lines[0]                                 # First line

# Read as single string
$Content = Get-Content "file.txt" -Raw

# Read specific lines
Get-Content "file.txt" -TotalCount 10     # First 10 lines
Get-Content "file.txt" -Tail 20          # Last 20 lines

# Follow file (like tail -f)
Get-Content "app.log" -Wait

# Read with encoding
Get-Content "unicode.txt" -Encoding UTF8

# Filter while reading
Get-Content "app.log" | Where-Object { $_ -match 'error' }

# Read binary file
$Bytes = [System.IO.File]::ReadAllBytes("file.bin")

Writing Files

üî∑ PowerShell - Writing Files 
# Overwrite file
Set-Content -Path "output.txt" -Value "Hello World"

# Write array (each element = line)
$Data = @("Line 1", "Line 2", "Line 3")
Set-Content "output.txt" $Data

# Append to file
Add-Content -Path "log.txt" -Value "$(Get-Date): New entry"

# Out-File (preserves formatting)
Get-Process | Out-File "processes.txt"
Get-Process | Out-File "processes.txt" -Append

# Redirect operators
Get-Service > "services.txt"              # Overwrite
Get-Service >> "services.txt"             # Append

# Specify encoding
Set-Content "utf8.txt" -Value $Data -Encoding UTF8
Out-File "output.txt" -Encoding ASCII

# Force create directories
Set-Content "C:\NewFolder\output.txt" -Value "Test" -Force

File Operations

üî∑ PowerShell - File Management 
# List files
Get-ChildItem "C:\Logs"
Get-ChildItem "C:\Logs" -Recurse           # Include subdirs
Get-ChildItem "C:\Logs" -Filter "*.log"    # Filter by extension

# Find files by criteria
Get-ChildItem -Recurse | Where-Object { $_.Length -gt 1MB }
Get-ChildItem -Recurse | Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) }

# Check if exists
if (Test-Path "C:\Config\app.conf") {
    Write-Host "File exists"
}

# Copy/Move/Remove
Copy-Item "source.txt" "destination.txt"
Copy-Item "C:\Logs\*" "C:\Backup\" -Recurse
Move-Item "old.txt" "new.txt"
Remove-Item "temp.txt"
Remove-Item "C:\Temp\*" -Recurse -Force

# Create directory
New-Item -Path "C:\NewFolder" -ItemType Directory

# Get file info
$File = Get-Item "app.log"
$File.Length                              # Size in bytes
$File.CreationTime
$File.LastWriteTime
$File.Extension

Working with CSV and JSON

üî∑ PowerShell - CSV and JSON 
# Import CSV
$Users = Import-Csv "users.csv"
$Users[0].Name                            # Access column
$Users | Where-Object Department -eq 'IT'

# Export CSV
Get-Process | Select-Object Name, CPU, Id | Export-Csv "processes.csv" -NoTypeInformation

# Import JSON
$Config = Get-Content "config.json" | ConvertFrom-Json
$Config.settings.timeout

# Export JSON
$Data = @{
    Name = "Server01"
    IP = "192.168.1.10"
    Roles = @("Web", "API")
}
$Data | ConvertTo-Json | Out-File "server.json"

# Deep JSON conversion
$ComplexData | ConvertTo-Json -Depth 5 | Out-File "deep.json"

# XML
[xml]$Xml = Get-Content "config.xml"
$Xml.configuration.appSettings.add

Path Operations

üî∑ PowerShell - Path Handling 
# Join paths safely
$FullPath = Join-Path "C:\Logs" "app.log"

# Split path components
Split-Path "C:\Logs\app.log" -Parent     # C:\Logs
Split-Path "C:\Logs\app.log" -Leaf       # app.log

# Get file extension
[System.IO.Path]::GetExtension("file.txt")  # .txt

# Change extension
[System.IO.Path]::ChangeExtension("file.txt", ".bak")

# Resolve full path
Resolve-Path ".\relative\path"

# Current directory
Get-Location
Set-Location "C:\Scripts"

# Temporary file
$TempFile = [System.IO.Path]::GetTempFileName()

Best Practices: File Operations
  • Use Join-Path: Handles path separators correctly
  • Test-Path first: Check existence before operations
  • Use -ErrorAction: Handle missing files gracefully
  • Specify encoding: Avoid character issues
  • Use -NoTypeInformation: Cleaner CSV exports
  • Large files: Stream with pipeline instead of -Raw
üíº
Interview Tips
  • Common Question: "How do you process a large log file efficiently?"
  • Strong Answer: Use Get-Content with pipeline filtering rather than -Raw to avoid loading entire file into memory. For example: Get-Content log.txt | Where-Object { $_ -match 'error' } | ForEach-Object { ... }. For very large files, use -ReadCount to batch process or [System.IO.StreamReader] for line-by-line reading.

Key Takeaways

  • Get-Content reads files; Set-Content writes/overwrites
  • Add-Content appends; Out-File preserves formatting
  • Use -Raw to read entire file as single string
  • Import-Csv/Export-Csv for CSV data
  • ConvertFrom-Json/ConvertTo-Json for JSON
  • Always use Join-Path for cross-platform compatibility
