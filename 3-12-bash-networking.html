Linux Network Tools for Security

Linux provides a rich set of networking tools that are essential for security professionals. From basic connectivity testing to packet capture and analysis, these tools form the foundation of network security operations.

üß†
Memory Aid: Core Network Tools
  • ip / ifconfig ‚Üí Interface configuration
  • ss / netstat ‚Üí Socket statistics, connections
  • ping / traceroute ‚Üí Connectivity testing
  • dig / nslookup ‚Üí DNS queries
  • curl / wget ‚Üí HTTP requests, downloads
  • nc (netcat) ‚Üí TCP/UDP connections, the "Swiss Army knife"
  • tcpdump / tshark ‚Üí Packet capture
  • nmap ‚Üí Port scanning, service detection

Network Configuration

üêß Bash - Network Configuration 
# Show all interfaces (modern)
ip addr show
ip a            # Short form

# Show specific interface
ip addr show eth0

# Show only IPv4 addresses
ip -4 addr

# Legacy ifconfig (may need net-tools package)
ifconfig
ifconfig eth0

# Show routing table
ip route show
route -n       # Legacy

# Show default gateway
ip route | grep default

# Show ARP cache (MAC to IP mappings)
ip neigh show
arp -a         # Legacy

# Show DNS servers
cat /etc/resolv.conf
resolvectl status

# Get public IP
curl -s ifconfig.me
curl -s ipinfo.io/ip
curl -s ipinfo.io/json | jq

Socket Statistics (ss/netstat)

üêß Bash - Socket Statistics 
# ss - Socket Statistics (modern, faster than netstat)

# All TCP connections
ss -t

# All listening ports
ss -l

# Listening TCP ports with process info
ss -tlnp

# All established connections
ss -t state established

# UDP sockets
ss -u

# All sockets with process names
ss -tunlp

# Filter by port
ss -t sport = :22           # Source port 22
ss -t dport = :443          # Destination port 443
ss -t '( dport = :80 or dport = :443 )'

# Filter by state
ss -t state time-wait
ss -t state close-wait      # Potential connection issues

# Count connections by state
ss -t | awk 'NR>1 {print $1}' | sort | uniq -c

# Legacy netstat (slower but familiar)
netstat -tulnp             # TCP/UDP listening with process
netstat -an | grep ESTABLISHED

Connectivity Testing

üêß Bash - Connectivity Tests 
# Basic ping
ping -c 4 google.com

# Ping with timestamp
ping -c 4 -D google.com

# Traceroute
traceroute google.com
traceroute -n google.com    # No DNS resolution (faster)

# MTR - combines ping and traceroute
mtr google.com
mtr --report -c 10 google.com  # Report mode

# Test specific port with nc (netcat)
nc -zv google.com 443
nc -zv -w 3 192.168.1.1 22   # 3 second timeout

# Test port range
nc -zv 192.168.1.1 20-25

# Quick TCP port test with bash
if timeout 1 bash -c " 2>/dev/null; then
    echo "Port open"
else
    echo "Port closed"
fi

# Check multiple hosts
for host in 8.8.8.8 1.1.1.1 9.9.9.9; do
    ping -c 1 -W 1 $host &>/dev/null && \
        echo "$host: UP" || echo "$host: DOWN"
done

DNS Queries

üêß Bash - DNS Queries 
# dig - DNS lookup utility (recommended)

# Basic A record lookup
dig google.com

# Short output
dig +short google.com

# Specific record types
dig google.com A           # IPv4
dig google.com AAAA        # IPv6
dig google.com MX          # Mail servers
dig google.com NS          # Name servers
dig google.com TXT         # TXT records (SPF, DKIM)
dig google.com SOA         # Start of Authority
dig google.com ANY         # All records

# Reverse DNS lookup
dig -x 8.8.8.8

# Use specific DNS server
dig @8.8.8.8 google.com
dig @1.1.1.1 google.com

# Trace DNS resolution path
dig +trace google.com

# nslookup (legacy but still useful)
nslookup google.com
nslookup google.com 8.8.8.8

# host command (simple output)
host google.com
host -t MX google.com

# Bulk DNS lookup
for domain in google.com amazon.com microsoft.com; do
    echo "$domain: $(dig +short $domain | head -1)"
done

HTTP Requests with curl

üêß Bash - curl HTTP Requests 
# Basic GET request
curl https://api.ipify.org?format=json

# Silent mode (no progress)
curl -s https://api.ipify.org?format=json

# Include response headers
curl -i https://httpbin.org/get

# Only headers
curl -I https://google.com

# Follow redirects
curl -L https://google.com

# POST request with JSON
curl -X POST https://httpbin.org/post \
    -H "Content-Type: application/json" \
    -d '{"name": "test", "value": 123}'

# POST with form data
curl -X POST https://httpbin.org/post \
    -d "username=admin&password=secret"

# Custom headers
curl -H "Authorization: Bearer TOKEN" \
    -H "Accept: application/json" \
    https://api.example.com/data

# Save output to file
curl -o output.html https://example.com
curl -O https://example.com/file.zip  # Keep original filename

# Show timing information
curl -w "DNS: %{time_namelookup}s\nConnect: %{time_connect}s\nTotal: %{time_total}s\n" \
    -o /dev/null -s https://google.com

# Verbose mode for debugging
curl -v https://example.com

# Ignore SSL certificate errors (testing only!)
curl -k https://self-signed.example.com

# Set timeout
curl --connect-timeout 5 --max-time 10 https://example.com

Netcat (nc) - The Swiss Army Knife

üêß Bash - Netcat Usage 
# Port scanning
nc -zv target.com 1-1000

# Banner grabbing
echo "" | nc -v target.com 22
nc -v target.com 80 <<< "HEAD / HTTP/1.0\r\n\r\n"

# Simple listener (for testing)
nc -lvp 4444

# Connect to listener
nc target.com 4444

# File transfer
# Receiver:
nc -lvp 4444 > received_file.txt
# Sender:
nc target.com 4444 < file_to_send.txt

# Simple chat
# Server: nc -lvp 4444
# Client: nc server_ip 4444

# UDP mode
nc -u target.com 53

# HTTP request via netcat
echo -e "GET / HTTP/1.1\r\nHost: example.com\r\n\r\n" | nc example.com 80

Port Scanning with Nmap

üêß Bash - Nmap Scanning 
# Basic TCP scan
nmap target.com

# Scan specific ports
nmap -p 22,80,443 target.com
nmap -p 1-1000 target.com
nmap -p- target.com         # All 65535 ports

# Fast scan (top 100 ports)
nmap -F target.com

# Service version detection
nmap -sV target.com

# OS detection
nmap -O target.com

# Aggressive scan (OS, version, scripts, traceroute)
nmap -A target.com

# UDP scan
sudo nmap -sU target.com

# Stealth SYN scan
sudo nmap -sS target.com

# Ping sweep (discover hosts)
nmap -sn 192.168.1.0/24

# Skip host discovery (assume online)
nmap -Pn target.com

# Output formats
nmap -oN normal.txt target.com      # Normal
nmap -oX output.xml target.com      # XML
nmap -oG grepable.txt target.com    # Grepable
nmap -oA all_formats target.com     # All formats

# Script scanning (NSE)
nmap --script=vuln target.com       # Vulnerability scripts
nmap --script=default target.com    # Default scripts
nmap --script=http-enum target.com  # Specific script

Security Reconnaissance Script

üêß Bash - Recon Script 
#!/bin/bash
# Quick reconnaissance script

TARGET="$1"

if [[ -z "$TARGET" ]]; then
    echo "Usage: $0 "
    exit 1
fi

echo "============================================"
echo "       RECONNAISSANCE: $TARGET"
echo "============================================"

# DNS Information
echo -e "\n[*] DNS Records"
echo "A Record:    $(dig +short $TARGET A | head -1)"
echo "MX Records:  $(dig +short $TARGET MX)"
echo "NS Records:  $(dig +short $TARGET NS)"

# WHOIS
echo -e "\n[*] WHOIS Info"
whois $TARGET 2>/dev/null | grep -iE "registrar|creation|expir|name server" | head -10

# HTTP Headers
echo -e "\n[*] HTTP Headers"
curl -sI https://$TARGET 2>/dev/null | head -15

# SSL Certificate
echo -e "\n[*] SSL Certificate"
echo | openssl s_client -connect $TARGET:443 2>/dev/null | \
    openssl x509 -noout -subject -dates 2>/dev/null

# Quick port scan
echo -e "\n[*] Common Ports"
for port in 21 22 23 25 53 80 110 143 443 445 3306 3389 8080; do
    timeout 1 bash -c " 2>/dev/null && \
        echo "  Port $port: OPEN"
done

echo -e "\n[*] Recon complete"

Packet Capture (tcpdump)

üêß Bash - tcpdump 
# Capture all traffic on interface
sudo tcpdump -i eth0

# Capture with verbose output
sudo tcpdump -v -i eth0

# Filter by host
sudo tcpdump host 192.168.1.100

# Filter by port
sudo tcpdump port 443
sudo tcpdump port 80 or port 443

# Filter by protocol
sudo tcpdump tcp
sudo tcpdump udp
sudo tcpdump icmp

# Capture to file
sudo tcpdump -w capture.pcap -i eth0

# Read from file
tcpdump -r capture.pcap

# Limit packet count
sudo tcpdump -c 100 -i eth0

# Show packet contents in ASCII
sudo tcpdump -A -i eth0 port 80

# Show packet contents in hex and ASCII
sudo tcpdump -XX -i eth0

# Complex filter: HTTP traffic to/from specific IP
sudo tcpdump 'host 192.168.1.100 and (port 80 or port 443)'

Interview Tips
  • Common Question: "How would you troubleshoot network connectivity issues in Linux?"
  • Strong Answer: Start with ping for basic connectivity, then traceroute to find where packets stop. Check ip addr for interface config, ip route for routing, ss -tunlp for open ports. Use dig for DNS issues. For deeper analysis, capture traffic with tcpdump.
  • Bonus: Mention checking firewall rules (iptables -L or ufw status) and system logs (/var/log/syslog)

Key Takeaways

  • Use ip (modern) over ifconfig (legacy) for interface config
  • Use ss (faster) over netstat for socket statistics
  • dig is the most powerful DNS tool; use +short for clean output
  • curl is essential for HTTP testing and API work
  • nc (netcat) is the Swiss Army knife for TCP/UDP operations
  • nmap is the standard for port scanning and service detection
  • Always use sudo for packet capture and raw socket operations
