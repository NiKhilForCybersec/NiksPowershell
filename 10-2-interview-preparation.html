Foundational Questions

Q: What makes PowerShell different from other shells?

Strong Answer:

PowerShell is object-orientedâ€”it passes .NET objects through the pipeline, not text. This means I can filter on actual property values like Where-Object CPU -gt 100 instead of parsing text with regex. It's built on .NET, giving access to the full .NET class library. It also has consistent Verb-Noun cmdlet naming and comprehensive built-in help with Get-Help.

Bonus Points: Mention that PowerShell 7 is cross-platform and open-source.

ğŸ’¼
Q: Explain the PowerShell pipeline.

Strong Answer:

The pipeline connects cmdlets using the | operator. Objects flow from left to right, with each cmdlet processing and potentially transforming them. Key pipeline cmdlets are Where-Object (filter), Select-Object (choose properties), ForEach-Object (process each), and Sort-Object (order results).

Pipeline binding works two ways: ByValue (object type matches parameter type) and ByPropertyName (object property names match parameter names). I always "filter left, format right" for performance.

Q: How do you handle errors in PowerShell?

Strong Answer:

ğŸ”· PowerShell - Error Handling Pattern 
# Two error types: Terminating and Non-Terminating

# Try/Catch for terminating errors
try {
    Get-Content "nonexistent.txt" -ErrorAction Stop
}
catch {
    Write-Error "Failed: $($_.Exception.Message)"
}
finally {
    # Cleanup code runs regardless
}

# -ErrorAction controls behavior per-cmdlet
# Stop, Continue, SilentlyContinue, Ignore

# $ErrorActionPreference sets default for session
$ErrorActionPreference = 'Stop'

Key Point: Mention the difference between terminating (stop execution) and non-terminating (continue) errors, and when to use -ErrorAction Stop.

Q: How would you find all failed login attempts in the last 24 hours?
ğŸ”· PowerShell - Failed Login Analysis 
# Method 1: Get-WinEvent with FilterHashtable (fastest)
$StartTime = (Get-Date).AddHours(-24)
Get-WinEvent -FilterHashtable @{
    LogName = 'Security'
    Id = 4625
    StartTime = $StartTime
} | ForEach-Object {
    [PSCustomObject]@{
        Time = $_.TimeCreated
        User = $_.Properties[5].Value
        SourceIP = $_.Properties[19].Value
        Reason = $_.Properties[8].Value
    }
} | Group-Object SourceIP | Sort-Object Count -Descending

# Why 4625? It's the Windows Security event for failed logon
# Properties array contains event-specific data

Bonus: Mention Event ID 4625 specifically, explain why FilterHashtable is faster than Where-Object, and note that Properties[19] contains the source IP for network logons.

Q: Write a script to find processes with network connections.
ğŸ”· PowerShell - Process Network Connections 
function Get-ProcessConnections {
    Get-NetTCPConnection -State Established | 
        ForEach-Object {
            $Process = Get-Process -Id $_.OwningProcess -ErrorAction SilentlyContinue
            [PSCustomObject]@{
                ProcessName = $Process.Name
                PID = $_.OwningProcess
                LocalPort = $_.LocalPort
                RemoteAddress = $_.RemoteAddress
                RemotePort = $_.RemotePort
                Path = $Process.Path
            }
        } | Sort-Object ProcessName
}

Get-ProcessConnections | Format-Table -AutoSize

Q: What's the difference between single and double quotes in Bash?

Strong Answer:

ğŸ§ Bash - Quote Differences 
NAME="World"

# Double quotes - variables ARE expanded
echo "Hello $NAME"    # Output: Hello World

# Single quotes - literal string, NO expansion
echo 'Hello $NAME'    # Output: Hello $NAME

# Use single quotes for regex patterns to prevent shell interpretation
grep '[0-9]+\.[0-9]+' file.txt

Key Point: Double quotes allow variable expansion and command substitution; single quotes preserve literal text. Use single quotes for regex to prevent shell interpretation.

ğŸ’¼
Q: How do you find the top 10 IPs in an Apache access log?
ğŸ§ Bash - Log Analysis 
# Apache log format: IP - - [date] "request" status size
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -10

# Breakdown:
# awk '{print $1}'  - Extract first field (IP)
# sort              - Sort alphabetically (required for uniq)
# uniq -c           - Count consecutive duplicates
# sort -rn          - Sort numerically, descending
# head -10          - Top 10 results

Key Point: Explain each command in the pipeline. Mention that sort before uniq is required because uniq only compares adjacent lines.

Q: Explain the difference between $* and $@ in Bash.
ğŸ§ Bash - Special Parameters 
#!/bin/bash
# Called with: ./script.sh "arg one" "arg two"

# $* - All args as single string
for arg in "$*"; do
    echo "$arg"
done
# Output: arg one arg two (one iteration)

# $@ - All args as separate strings (PREFERRED)
for arg in "$@"; do
    echo "$arg"
done
# Output: arg one (first iteration)
#         arg two (second iteration)

# Always use "$@" to preserve argument boundaries

ğŸ”’ Security Scenario Questions

ğŸ’¼
Q: Walk me through how you'd respond to a malware alert using scripts.

Strong Answer Structure (use STAR method):

  1. Collect Volatile Data First - Processes, network connections, logged-on users, memory (order of volatility: R.A.N.P.D.)
  2. Isolate if Needed - Use EDR API to network-isolate the host while preserving data
  3. Collect Persistent Artifacts - Event logs, registry run keys, scheduled tasks, prefetch files
  4. Hash Everything - SHA256 all collected files for chain of custody
  5. Package and Document - Create manifest, timestamp, compress with case number
ğŸ”· PowerShell - Quick IR Collection 
# Quick volatile data collection
$IRData = @{
    Processes = Get-CimInstance Win32_Process | Select Name, ProcessId, CommandLine
    Connections = Get-NetTCPConnection -State Established
    Users = quser 2>$null
    Timestamp = Get-Date
}
$IRData | ConvertTo-Json -Depth 5 | Out-File "IR_$(Get-Date -f yyyyMMdd_HHmmss).json"

Q: How would you automate user offboarding from a security perspective?

Strong Answer:

ğŸ”· PowerShell - Secure Offboarding 
function Invoke-SecureOffboarding {
    param([string]$Username)
    
    # 1. Disable account immediately (don't delete - preserve for forensics)
    Disable-ADAccount -Identity $Username
    
    # 2. Reset password to random value
    $RandomPwd = [System.Web.Security.Membership]::GeneratePassword(32,8)
    Set-ADAccountPassword -Identity $Username -Reset -NewPassword (ConvertTo-SecureString $RandomPwd -AsPlainText -Force)
    
    # 3. Remove from all groups (except Domain Users)
    Get-ADUser $Username -Properties MemberOf | 
        Select-Object -ExpandProperty MemberOf |
        ForEach-Object { Remove-ADGroupMember $_ $Username -Confirm:$false }
    
    # 4. Revoke Azure/M365 sessions
    Revoke-AzureADUserAllRefreshToken -ObjectId $Username
    
    # 5. Move to Disabled Users OU
    Move-ADObject -Identity (Get-ADUser $Username).DistinguishedName -TargetPath "OU=Disabled Users,DC=corp,DC=local"
    
    # 6. Log the action
    Write-EventLog -LogName Application -Source "IT-Automation" -EventId 1001 -Message "User $Username offboarded by $env:USERNAME"
}

Key Points: Disable don't delete (audit trail), reset password, revoke sessions, remove group memberships, move to Disabled OU, and log everything.

Best Practice: Live Coding Strategy
  1. Clarify Requirements - Ask about input/output format, edge cases, error handling expectations
  2. Think Aloud - Verbalize your approach before coding: "I'll start by..."
  3. Start Simple - Get a working solution first, then optimize
  4. Use Comments - Write pseudocode comments, then fill in code
  5. Test Incrementally - Run small pieces as you go
  6. Handle Errors - Add basic error handling even if not asked
âš ï¸
Common Live Coding Mistakes
  • Going silent - Keep talking! Silence is uncomfortable for interviewers
  • Jumping to code - Take 30 seconds to plan your approach
  • Ignoring edge cases - What if the file is empty? What if input is null?
  • Over-engineering - Don't build a framework when a function will do
  • Not testing - Run your code with sample input before saying "done"
