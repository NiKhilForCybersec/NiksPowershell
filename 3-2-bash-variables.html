Bash Variables Basics

Bash variables are untyped by default‚Äîeverything is a string. Understanding how to properly declare, quote, and expand variables is essential for writing reliable scripts.

üß†
Memory Aid: Variable Rules
  • No spaces around = ‚Üí name="value" ‚úì name = "value" ‚úó
  • $ to read, no $ to assign ‚Üí name="John" then echo $name
  • Double quotes expand ‚Üí "Hello $name" ‚Üí "Hello John"
  • Single quotes literal ‚Üí 'Hello $name' ‚Üí "Hello $name"
  • Curly braces for clarity ‚Üí ${name} prevents ambiguity

Variable Declaration

üêß Bash - Variable Basics 
#!/bin/bash

# Variable assignment (NO spaces around =)
name="John"
count=42
path=/var/log

# WRONG - these cause errors
# name = "John"   # Tries to run 'name' command
# name= "John"    # Sets empty, then runs 'John'

# Read variable with $
echo $name
echo "Hello, $name"

# Curly braces for clarity
file="report"
echo "${file}_backup.txt"   # report_backup.txt
echo "$file_backup.txt"     # Error - $file_backup doesn't exist

# Read-only variables
readonly PI=3.14159
# PI=3.14  # Error: PI is read-only

# Unset a variable
temp="temporary"
unset temp

# Check if variable is set
if [[ -z "$undefined_var" ]]; then
    echo "Variable is empty or unset"
fi

if [[ -n "$name" ]]; then
    echo "Variable has a value"
fi

Quoting Rules

üêß Bash - Quoting 
#!/bin/bash

name="Alice"
price='$100'

# Double quotes: Variables and commands expand
echo "Hello, $name"           # Hello, Alice
echo "Today is $(date +%A)"   # Today is Monday
echo "Home: $HOME"             # Home: /home/user

# Single quotes: Everything is literal
echo 'Hello, $name'           # Hello, $name
echo 'Price: $100'            # Price: $100
echo 'Command: $(whoami)'     # Command: $(whoami)

# No quotes: Word splitting and globbing occur
files="file1.txt file2.txt"
echo $files                    # Each file becomes separate argument
echo "$files"                  # Preserved as single string

# Escape special characters
echo "She said \"Hello\""
echo "Path: \$HOME"            # Path: $HOME (literal)
echo "Line1\nLine2"            # \n is literal
echo -e "Line1\nLine2"         # \n is newline

# Best practice: ALWAYS quote variables
filename="my file.txt"
cat "$filename"               # Correct
# cat $filename               # WRONG - becomes: cat my file.txt

# Mixing quotes
echo "It's a nice day"        # Works - single quote inside double
echo 'Say "Hello"'            # Works - double quote inside single

String Operations

üêß Bash - String Manipulation 
#!/bin/bash

str="Hello World"

# String length
echo "${#str}"                # 11

# Substring extraction: ${string:position:length}
echo "${str:0:5}"             # Hello (first 5 chars)
echo "${str:6}"               # World (from position 6)
echo "${str: -5}"             # World (last 5 chars, note space)

# String replacement
path="/home/user/documents/file.txt"
echo "${path/user/admin}"     # Replace first occurrence
echo "${path//\//-}"          # Replace all / with - (// = global)

# Remove pattern from beginning/end
file="document.tar.gz"
echo "${file#*.}"             # tar.gz (remove shortest from start)
echo "${file##*.}"            # gz (remove longest from start)
echo "${file%.*}"             # document.tar (remove shortest from end)
echo "${file%%.*}"            # document (remove longest from end)

# Case conversion (Bash 4+)
name="JoHn DoE"
echo "${name,,}"              # john doe (lowercase)
echo "${name^^}"              # JOHN DOE (uppercase)
echo "${name^}"               # John doe (capitalize first)

# String concatenation
first="Hello"
second="World"
full="$first $second"
full+="!"                      # Append
echo "$full"                   # Hello World!

Default Values and Substitution

üêß Bash - Parameter Expansion 
#!/bin/bash

# Default value if unset or empty
name=""
echo "${name:-default}"       # "default" (doesn't modify $name)

# Default value if unset only (empty string is okay)
echo "${name-default}"        # "" (name is set, just empty)

# Assign default if unset or empty
echo "${name:=guest}"         # "guest" (also sets $name)
echo "$name"                   # "guest"

# Error if unset or empty
# ${name:?Error message}      # Exits with error if unset/empty

# Use alternative value if set
value="exists"
echo "${value:+was set}"      # "was set" (because value exists)
echo "${unset:+was set}"      # "" (because unset)

# Common patterns
# Use environment variable or default
LOG_LEVEL="${LOG_LEVEL:-INFO}"
CONFIG_PATH="${CONFIG_PATH:-/etc/myapp/config.yml}"

# Require variable to be set
: "${API_KEY:?API_KEY must be set}"

# Check and set in one line
TIMEOUT="${TIMEOUT:-30}"

Arrays

üêß Bash - Arrays 
#!/bin/bash

# Indexed array declaration
servers=("web01" "web02" "db01" "db02")

# Access elements (0-indexed)
echo "${servers[0]}"          # web01
echo "${servers[2]}"          # db01
echo "${servers[-1]}"         # db02 (last element)

# All elements
echo "${servers[@]}"          # All as separate words
echo "${servers[*]}"          # All as single string

# Array length
echo "${#servers[@]}"         # 4

# Array indices
echo "${!servers[@]}"         # 0 1 2 3

# Add element
servers+=("cache01")

# Slice array
echo "${servers[@]:1:2}"      # web02 db01 (from index 1, 2 elements)

# Loop through array
for server in "${servers[@]}"; do
    echo "Processing: $server"
done

# Loop with index
for i in "${!servers[@]}"; do
    echo "$i: ${servers[$i]}"
done

# Associative array (Bash 4+)
declare -A config
config[host]="localhost"
config[port]="8080"
config[timeout]="30"

echo "Host: ${config[host]}"
echo "Keys: ${!config[@]}"     # host port timeout
echo "Values: ${config[@]}"   # localhost 8080 30

# Check if key exists
if [[ -v config[host] ]]; then
    echo "Host is configured"
fi

Numbers and Arithmetic

üêß Bash - Arithmetic 
#!/bin/bash

# Arithmetic expansion $(( ))
a=10
b=3

echo "Sum: $((a + b))"        # 13
echo "Diff: $((a - b))"       # 7
echo "Product: $((a * b))"    # 30
echo "Quotient: $((a / b))"   # 3 (integer division)
echo "Remainder: $((a % b))"  # 1
echo "Power: $((a ** 2))"     # 100

# Increment/decrement
count=0
(( count++ ))                   # count = 1
(( count += 5 ))                # count = 6
(( count-- ))                   # count = 5

# let command
let "result = a + b"
let "a++"

# expr command (older, portable)
result=$(expr $a + $b)

# Floating point with bc
echo "10 / 3" | bc -l         # 3.33333...
result=$(echo "scale=2; 10/3" | bc)  # 3.33

# Comparison in arithmetic context
if (( a > b )); then
    echo "$a is greater than $b"
fi

# Ternary operator
max=$(( a > b ? a : b ))

# Hex and octal
hex=0xFF
echo "Hex $hex = $((hex))"    # 255
oct=0777
echo "Octal $oct = $((oct))"  # 511

Special Variables

üêß Bash - Special Variables 
#!/bin/bash

# Script arguments
echo "Script name: $0"
echo "First arg: $1"
echo "Second arg: $2"
echo "All args: $@"            # As separate words
echo "All args: $*"            # As single string
echo "Number of args: $#"

# Process info
echo "Process ID: $$"
echo "Parent PID: $PPID"

# Exit status
ls /nonexistent 2>/dev/null
echo "Exit status: $?"         # Non-zero (error)

ls / >/dev/null
echo "Exit status: $?"         # 0 (success)

# Background process PID
sleep 10 &
echo "Background PID: $!"

# Environment variables
echo "Home: $HOME"
echo "User: $USER"
echo "Shell: $SHELL"
echo "Path: $PATH"
echo "PWD: $PWD"
echo "Hostname: $HOSTNAME"

# Random number
echo "Random: $RANDOM"         # 0-32767
echo "Random 1-100: $((RANDOM % 100 + 1))"

# Seconds since script started
echo "Seconds: $SECONDS"

# Line number in script
echo "Line: $LINENO"

Command Substitution

üêß Bash - Command Substitution 
#!/bin/bash

# Modern syntax: $(command)
today=$(date +%Y-%m-%d)
user=$(whoami)
files=$(ls -1 | wc -l)

echo "Today: $today"
echo "User: $user"
echo "Files: $files"

# Nested command substitution
echo "Home dir contains $(ls $(echo $HOME) | wc -l) items"

# Backtick syntax (older, avoid)
today=`date +%Y-%m-%d`

# Process substitution
# Compare two command outputs
diff <(ls dir1) <(ls dir2)

# Read from command
while read -r line; do
    echo "Line: $line"
done < <(cat /etc/passwd | head -5)

# Security use case: Get current IP
my_ip=$(curl -s ifconfig.me)
echo "External IP: $my_ip"

Best Practices: Bash Variables
  • Always quote variables: "$var" prevents word splitting
  • Use ${} for clarity: ${var}_suffix is unambiguous
  • Check if set: Use ${var:-default} for defaults
  • Use local in functions: Prevents variable leakage
  • Use readonly for constants: readonly CONFIG="/etc/app"
  • Prefer $(command) over backticks: Nestable and clearer
üíº
Interview Tips
  • Common Question: "What's the difference between single and double quotes in Bash?"
  • Strong Answer: Double quotes allow variable expansion and command substitution ("Hello $name" ‚Üí "Hello John"), while single quotes preserve everything literally ('$name' ‚Üí "$name"). Always use double quotes around variables to prevent word splitting on spaces.

Key Takeaways

  • No spaces around = in assignments
  • Double quotes expand variables, single quotes don't
  • Always quote variables to prevent word splitting
  • Use ${var:-default} for default values
  • Arrays use parentheses: arr=(a b c)
  • Arithmetic uses $(( )) or (( ))
  • $? is exit status, $$ is PID, $@ is all arguments
